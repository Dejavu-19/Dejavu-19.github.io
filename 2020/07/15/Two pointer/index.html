Unsolved Longest Substring with Same Letters after Replacement (hard)‚ú®

Given a string with lowercase letters only, if you are allowed to replace no more than ‚Äòk‚Äô letters with any letter, find the length of the longest substring having the same letters after replacement.

Solution1:

  1. We‚Äôll iterate through the string to add one letter at a time in the window.

  2. We‚Äôll also keep track of the count of the maximum repeating letter in any window (let‚Äôs call it maxRepeatLetterCount).

  3. So, at any time, we know that we can have a window which has one letter repeating maxRepeatLetterCount times; this means we should try to replace the remaining letters.

  4. If we have more than ‚Äòk‚Äô remaining letters, we should shrink the window as we are not allowed to replace more than ‚Äòk‚Äô letters.

  5. Time Complexity

    The above algorithm‚Äôs time complexity will be O(N)O(N), where ‚ÄòN‚Äô is the number of letters in the input string.

    Space Complexity

    As we expect only the lower case letters in the input string, we can conclude that the space complexity will be O(26)O(26) to store each letter‚Äôs frequency in the HashMap, which is asymptotically equal to O(1)O(1).

Given a string and a list of words, find all the starting indices of substrings in the given string that are a concatenation of all the given words exactly once without any overlapping of words. It is given that all words are of the same length.

Solution1:

  1. Keep the frequency of every word in a HashMap.
  2. Starting from every index in the string, try to match all the words.
  3. In each iteration, keep track of all the words that we have already seen in another HashMap.
  4. If a word is not found or has a higher frequency than required, we can move on to the next character in the string.
  5. Store the index if we have found all the words.
